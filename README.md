# Прогнозирование оттока клиентов

## Описание проекта
Цель данного проекта — построение модели машинного обучения для предсказания оттока клиентов телекоммуникационной компании. Помимо технического сравнения классификаторов, ключевой частью работы является интерпретация результатов с помощью SHAP-анализа для формирования бизнес-рекомендаций.

**Ключевые темы:**
*   Классификация и работа с категориальными признаками.
*   Сравнение моделей: Logistic Regression, XGBoost, Random Forest.
*   SHAP-анализ и оптимизация порога классификации.

## Dataset
Используется датасет **Telco Customer Churn** с платформы Kaggle.
*   **Целевая переменная:** `Churn`.
*   **Особенности данных:** Присутствует дисбаланс классов (доля оттока ~26.5%), что требует использования метрик F1-score и ROC-AUC вместо Accuracy.

## Этапы решения

### 1. Загрузка данных и EDA
Проведен первичный анализ данных:
*   Обнаружены пропуски в признаке `TotalCharges` (заполнены медианой).
*   Выявлена высокая корреляция между стажем клиента (`tenure`) и общей суммой платежей (`TotalCharges`).
*   **Визуализация:** Показано, что клиенты с помесячной оплатой (`Month-to-month`) и сроком обслуживания менее 12 месяцев уходят значительно чаще.

### 2. Предобработка
*   **Кодирование:** Применено One-Hot Encoding для всех категориальных признаков.
*   **Масштабирование:** Использован `StandardScaler` для признакового пространства (критично для логистической регрессии).
*   **Разделение:** Данные разделены на обучающую и тестовую выборки (70/30) с сохранением пропорций классов (`stratify=y`).

### 3. Обучение и сравнение моделей
Были обучены и протестированы три модели:
1.  **Logistic Regression** (Логистическая регрессия)
2.  **Random Forest** (Случайный лес)
3.  **XGBoost** (Градиентный бустинг)

**Результаты сравнения:**
Лучший результат показала **Logistic Regression**:
*   **F1 Score:** ~0.61
*   **ROC-AUC:** ~0.84

*Примечание:* Ансамблевые методы (RF, XGBoost) показали чуть худший результат из-за большого количества разреженных признаков после One-Hot Encoding относительно небольшого объема данных.

### 4. Оптимизация порога классификации
Стандартный порог 0.5 не всегда оптимален. С помощью анализа кривой Precision-Recall был подобран оптимальный порог отсечения (**~0.29**). Это позволило улучшить баланс между Precision и Recall и повысить итоговый F1-score.

### 5. SHAP-анализ и бизнес-интерпретация
Для лучшей модели (Logistic Regression) проведен SHAP-анализ важности признаков.

**Топ-5 влияющих признаков:**
1.  `tenure` (Срок обслуживания) — чем меньше срок, тем выше риск ухода.
2.  `InternetService_Fiber optic` (Оптоволоконный интернет) — клиенты с этим типом интернета уходят чаще.
3.  `MonthlyCharges` (Ежемесячная плата) — высокая плата коррелирует с оттоком.
4.  `Contract_Two year` (Контракт на 2 года) — сильный отрицательный фактор (удерживает клиентов).
5.  `TotalCharges` (Общая сумма платежей).
